# EL6851 with CX5120

This sample explains how to write a simple PLC program for DMX in TwinCAT and how to link it with the hardware. Search for DMX devices.

Sample: [DMX\_Sample\_EL6851\_CX5120.zip](https://infosys.beckhoff.com/content/1033/tcplclib_tc2_dmx/Resources/zip/6202185099.zip)

## Hardware

## Setting up the components

The following hardware is required:

* 1x CX5120 Embedded PC
* 1x EL6851 DMX Master terminal
* 1x EL9011 end cap

Set up the hardware and the DMX components as described in the respective documents.

## Software

## Creation of the PLC program

Create a new "TwinCAT XAE project" and a "Standard PLC project".

Add the library Tc2\_DMX in the PLC project under "References".

Generate a global variable list with the name GVL\_EIB and create the following variables:

```
VAR_GLOBAL  
    stEL6851InData      AT %I* : ST_EL6851InDataEx;  
    stKL6851OutData     AT %Q* : ST_EL6851OutData;  
    stCommandBuffer            : ST_DMXCommandBuffer;  
END_VAR
```

**stEL6851InData :** Input variable for the DMX terminal (see [ST\_EL6851InDataEx](ms-xhelp:///?Id=beckhoff-a226-455d-b033-54541a8350cb)).

**stEL6851OutData :** Output variable for the DMX terminal (see [ST\_EL6851OutData](ms-xhelp:///?Id=beckhoff-328e-47de-9cc1-599ac43987c4)).

**stCommandBuffer:** Required for the communication with DMX (see [ST\_DMXCommandBuffer](ms-xhelp:///?Id=beckhoff-b046-4e82-a9e9-879db0644a35)).

Then create a program (CFC) for background communication with DMX. The function block [FB\_EL6851CommunicationEx](ms-xhelp:///?Id=beckhoff-6a9b-4d50-8a01-d179a95958e3) is called in this program. Make sure to link the communication function block with *stEL6851InData*, *stEL6851OutData* and *stCommandBuffer*.

![21768161](/tcplclib_tc2_dmx/1033/Images/gif/6201515787__Web.gif)

Create a MAIN program (CFC) in which the function block [FB\_DMXDiscovery](ms-xhelp:///?Id=beckhoff-0635-4877-88b6-fa9af45fe6c6) is called up. The input *stCommandBuffer* of the function block is linked with the global variable *stCommandBuffer*. If the variable *bStart* is TRUE, the search for DMX devices is started. The number of found devices is stored in variable *iFoundDevices* and additional information in *arrDMXDeviceInfoList*.

![61695723](/tcplclib_tc2_dmx/1033/Images/gif/6201512843__Web.gif)

Navigate to the task configuration section and configure the PlcTask. By way of example, the task is assigned priority 16 and a cycle time of 6 ms.

![18390601](/tcplclib_tc2_dmx/1033/Images/gif/6202175627__Web.gif)

Create a further task for the background communication. Assign a higher priority (smaller number) and a lower interval time to this task than the PlcTask.

![31297682](/tcplclib_tc2_dmx/1033/Images/gif/6201509899__Web.gif)

Add the program for the communication to this task. Further information on the task configuration can be found in the description of the function block [FB\_EL6851CommunicationEx](ms-xhelp:///?Id=beckhoff-6a9b-4d50-8a01-d179a95958e3).

![13243689](/tcplclib_tc2_dmx/1033/Images/gif/6202178699__Web.gif)

## I/O configuration

Select the CX as target system and initiate a search for its hardware.

Make the DMX inputs 1 to 64 available by opening the *Process data* tab of the EL6851 and selecting the option *0x1A01* under *Inputs*.

![52084342](/tcplclib_tc2_dmx/1033/Images/gif/6201518731__Web.gif)

In the project instance within the PLC section, you can see that the input and output variables are assigned to the corresponding tasks.

Link the global variables of the PLC program with the inputs and outputs of the EtherCAT Terminal.

![66105898](/tcplclib_tc2_dmx/1033/Images/gif/6202181771__Web.gif)

Create the Solution and enable the configuration.
