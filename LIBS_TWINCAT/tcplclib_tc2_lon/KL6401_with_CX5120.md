# KL6401 with CX5120

This example explains how to write a simple PLC program for LON in TwinCAT and how to link it with the hardware. The task is to change the state of a switching output with a button.

Example: [LON\_Sample\_KL6401\_CX5120.zip](https://infosys.beckhoff.com/content/1033/tcplclib_tc2_lon/Resources/zip/6164908043.zip)

## Hardware

## Setting up the components

* 1x CX5120 Embedded PC
* 1x KL1104 four-channel digital input terminal (for the Set/Reset function)
* 1x KL6401 LON terminal
* 1x KL9010 end terminal

Set up the hardware and the LON components as described in the respective documents.

This example assumes that a Set button was linked to the first KL1104 input and a Reset button to the second.

## Software

## Creation of the PLC program

Create a new “TwinCAT XAE project” and a “Standard PLC project”.  
Add the library Tc2\_LON in the PLC project under “References”.  
Generate a global variable list with the name GVL\_LON and create the following variables:

```
VAR_GLOBAL  
    bSet                AT %I* : BOOL;  
    bReset              AT %I* : BOOL  
    stParameter_IN      AT %I* : ST_LON_Parameter_IN_36B;  
    stParameter_OUT     AT %Q* : ST_LON_Parameter_OUT_36B;  
    stLON_Com                  : ST_LON_Communication  
END_VAR
```

**bSet:** Input variable for the Set button.

**bReset:** Input variable for the Reset button.

**stParameter\_IN:** Input structure for the LON terminal (see [ST\_LON\_Parameter\_IN\_36B](ms-xhelp:///?Id=beckhoff-0ba5-4029-a0b0-b0c0cf8a1688)).

**stParameter\_OUT:** Output structures for the LON terminal (see [ST\_LON\_Parameter\_OUT\_36B](ms-xhelp:///?Id=beckhoff-6069-47e0-ba59-c455c7544910)).

**stLON\_Com:** Structure for the internal communication with LON (see [ST\_LON\_Communication](ms-xhelp:///?Id=beckhoff-2ad7-4b6a-944d-a82a3e3847ef)).

All LON function blocks must be called in the same task.

Create a LON program (CFC) in which the function blocks [FB\_LON\_KL6401](ms-xhelp:///?Id=beckhoff-cce2-44f9-8642-d620dc9ab975) and [FB\_SEND\_095\_SNVT\_switch](ms-xhelp:///?Id=beckhoff-309b-4e0d-84c4-4d1ca61f9111) are called up. At the communication block link the variables *stParameter\_IN*, *stParameter\_Out* and *stLON\_Com*.

![21484299](/tcplclib_tc2_lon/1033/Images/gif/6164515595__Web.gif)

Link the local variable *bSwitch* with the global variables *bSet* and *bReset*, then with the selected input. Link the local variable *stValue* (see [ST\_LON\_SNVT\_switch](ms-xhelp:///?Id=beckhoff-84a7-4beb-828d-ef824dbb072b)) with the selected output, then with the input *stValue* of the send block.

![59140967](/tcplclib_tc2_lon/1033/Images/gif/6164518539__Web.gif)

Go to the task configuration and give the task a lower interval time.

Further conditions can be found in the description of the function block [FB\_LON\_KL6401](ms-xhelp:///?Id=beckhoff-cce2-44f9-8642-d620dc9ab975).

![62506663](/tcplclib_tc2_lon/1033/Images/gif/6164900491__en-US__Web.gif)

## I/O configuration

Select the CX as target system and initiate a search for its hardware. In the project instance within the PLC section, you can see that the input and output variables are assigned to the input and output variables of the task.

Now link the variables with the inputs and outputs of the Bus Terminals.

The linking of the LON variables is described in detail below.

Right-click the variable *wParameterStatus* of the input structure and select "Change link".

![25689059](/tcplclib_tc2_lon/1033/Images/gif/6164521483__Web.gif)

Select the terminal under "I/O Configuration", click "Parameter Status" and confirm with "OK".

![29874939](/tcplclib_tc2_lon/1033/Images/gif/6164524427__Web.gif)

In the next step, right-click the structure *stParameterReadValue* within the input structure and select "Change Link".

*wDummy* is not linked.

![439002](/tcplclib_tc2_lon/1033/Images/gif/6164872971__Web.gif)

Select "All Types" and "Continuous", then select "Parameter Input Data 1" to "Parameter Input Data 8" with the left mouse button and the >SHIFT< key. Then click "OK".

![3951024](/tcplclib_tc2_lon/1033/Images/gif/6164875915__Web.gif)

Now also link the variables *byParameterType* and *byLONStatus*.

![35559222](/tcplclib_tc2_lon/1033/Images/gif/6164878859__Web.gif)![51597550](/tcplclib_tc2_lon/1033/Images/gif/6164881803__Web.gif)

You can now check the connection. To this end, select the KL6401 in the "I/O Configuration" and open it. All terminal data should now show a small arrow. If that is the case, then proceed in exactly the same way with the outputs.

![61724767](/tcplclib_tc2_lon/1033/Images/gif/6164897547__Web.gif)
