# MC\_AxUtiStandardInit\_BkPlcMc (from V3.0)

![49774232](/tf5810_tc3_hydraulic_positioning/1033/Images/gif/9007200854822283__Web.gif)

The function block handles the initialization and monitoring of axis components.

```
VAR_INPUT  
    AxisName:           STRING(255);  
    PathName:           STRING(255);  
    pDeviceInput:       POINTER TO ST_TcPlcDeviceInput:=0;  
    pDeviceOutput:      POINTER TO ST_TcPlcDeviceOutput:=0;  
    pLogBuffer:         POINTER TO ST_TcPlcMcLogBuffer:=0;  
    pStAxAuxLabels:     POINTER TO ST_TcMcAuxDataLabels:=0;  
    pStAxAutoParams:    POINTER TO ST_TcMcAutoIdent;  
    pStAxCommandBuf:    POINTER TO ST_TcPlcCmdBuffer_BkPlcMc:=0;    (* ab/from V3.0.8 *)  
    nLogLevel:          DINT:=0;  
END_VAR
```

```
VAR_OUTPUT  
    Busy:       BOOL;  
    Done:       BOOL;  
    Ready:      BOOL;  
    Error:      BOOL;  
    ErrorID:    UDINT;  
END_VAR
```

```
VAR_INOUT  
    Axis:       Axis_Ref_BkPlcMc;  
END_VAR
```

**AxisName:** Here, the text-based name of the axis (example: 'Axis\_1') should be transferred.

**PathName:** Here, the text-based path name (example: 'C:\TwinCAT\Project\'), under which the axis parameters are to be saved, should be transferred.

**pDeviceInput:** This parameter is used to transfer the address of a variable of type [ST\_TcPlcDeviceInput](ms-xhelp:///?Id=beckhoff-dec6-4dd6-a155-f200065a8d2d).

**pDeviceOutput:** This parameter is used to transfer the address of a variable of type [ST\_TcPlcDeviceOutput](ms-xhelp:///?Id=beckhoff-fec4-45f2-b458-d8180069b32f).

**pLogBuffer:** Here, the address of a variable of type [ST\_TcPlcMcLogBuffer](ms-xhelp:///?Id=beckhoff-2e65-4b91-955d-12cfc3c231b1) can be transferred.

**pAuxLabels:** Here, the address of an [ST\_TcMcAuxDataLabels](ms-xhelp:///?Id=beckhoff-50c8-4b5f-bdf7-0e14cb1fc422) structure with label texts for customer-specific axis parameters can be transferred.

**pStAxAutoParams**: Here, the address of a variable of type [ST\_TcMcAutoIdent](ms-xhelp:///?Id=beckhoff-d8cd-4957-b2b8-b8cd53f4d9b3) can be transferred.

**pStAxCommandBuf**: From V3.0.8, the input **BufferMode** is available in various function blocks, as defined by PLCopen. The functionality that can be controlled with this is currently in preparation. In this context this command buffer was amended.

|  |  |
| --- | --- |
| 45314905 | The input **pStAxCommandBuf** must currently not be supplied, or only with the value 0. |

**nLogLevel:** Here, a [coded value](ms-xhelp:///?Id=beckhoff-d319-45ac-96c4-54e861345d4c) should be transferred, which specifies the threshold value for recording of messages.

**Error**: The occurrence of an error is indicated here.

**ErrorID**: An encoded indication of the cause of the error is provided here.

**Axis**: Here, the address of a variable of type [Axis\_Ref\_BkPlcMc](ms-xhelp:///?Id=beckhoff-002b-4aba-81ea-1693fb03013c) should be transferred.

## Behaviour of the function block

With each call the function block examines the transferred axis interface and the transferred pointers. If a change is detected, the function block indicates in the transferred [Axis\_Ref\_BkPlcMc](ms-xhelp:///?Id=beckhoff-002b-4aba-81ea-1693fb03013c) structure that the axis has to be reinitialised. The [MC\_AxParamLoad\_BkPlcMc](ms-xhelp:///?Id=beckhoff-92c8-44d1-ae87-dd14f51905c7) function block used by the function block will now automatically load the axis parameters from the file. If **pAuxLabels** is supplied, an [MC\_AxParamAuxLabelsLoad\_BkPlcMc](ms-xhelp:///?Id=beckhoff-3d0d-45fe-ad0c-bbd985c13a84) function block is then used to load the label texts for the customer-specific axis parameters.

| 5180964 | The strings transferred as AxisName and PathName must not contain spaces or special characters, which would make them unsuitable for generating a file name. The file name is generated by concatenating the transferred strings and adding the extension '.dat'. The file name for the label texts of the customer-specific axis parameter is generated in the same way, but with the extension '.txt'. |
