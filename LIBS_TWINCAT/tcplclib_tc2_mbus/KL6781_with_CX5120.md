# KL6781 with CX5120

This example describes how a simple PLC program for M-Bus can be written in TwinCAT and how it is linked with the hardware. The task is to read a counter with four digital inputs.

Example: [MBus\_Sample\_KL6781\_CX5120.zip](https://infosys.beckhoff.com/content/1033/tcplclib_tc2_mbus/Resources/zip/6218378891.zip)

## Hardware

## Setting up the components

The following hardware is required:

* 1x CX5120 Embedded PC
* 1x KL6781 M-Bus master terminal
* 1x KL9010 end terminal

Set up the hardware and the M-Bus components as described in the associated documentation.

This example assumes that the counter address is known.

## Software

## Creation of the PLC program

Create a new "TwinCAT XAE project" and a "Standard PLC project".

Add the library Tc2\_MBus under **References** in the PLC project.

Generate a Global Variable List with the name GVL\_MBus and create the following variables:

```
VAR_GLOBAL  
    stComIn    AT %I* : ST_KL6781inData22B;  
    stComOut   AT %Q* : ST_KL6781outData22B;  
    stCom             : ST_MBUS_Communication;  
END_VAR
```

**stComIn:** Input variable for the M-Bus terminal (see [ST\_KL6781inData22B](ms-xhelp:///?Id=beckhoff-3864-4b56-a8e1-e0e0bc651d5b)).

**stComOut:** Output variable for the M-Bus terminal (see [ST\_KL6781outData22B](ms-xhelp:///?Id=beckhoff-acb7-4dc6-ba15-ec9ae0ee390e))

**stCom:** Required for the communication with M-Bus (see [ST\_MBUS\_Communication](ms-xhelp:///?Id=beckhoff-c554-4dbc-b2f4-dac10e7c746d)).

Create a program (CFC) for the background communication with M-Bus. The [FB\_MBUSKL6781](ms-xhelp:///?Id=beckhoff-2b5d-4be4-b4d2-353673675260) block is called in this program. Make sure to link the communication block with stComIn, stComOut and stCom.

![35960896](/tcplclib_tc2_mbus/1033/Images/gif/6218346251__Web.gif)

Create a MAIN program (CFC) in which the block [FB\_MBUS\_REL\_PadIn4](ms-xhelp:///?Id=beckhoff-8ecb-427d-a02b-f593c8a75091) is called up. Link the input *usiAddress* of the counter block with the local variable *usiAddress* and *stCom* with the global variable *stCom*.

![55212610](/tcplclib_tc2_mbus/1033/Images/gif/6218349195__Web.gif)

Navigate to the task configuration section and configure the PlcTask. By way of example, the task is assigned priority 16 and a cycle time of 6 ms.

![27151449](/tcplclib_tc2_mbus/1033/Images/gif/6218352139__Web.gif)

Create a further task for the background communication. Assign a higher priority (smaller number) and a lower interval time to this task than the PLCTask.

![43036453](/tcplclib_tc2_mbus/1033/Images/gif/6217533963__Web.gif)

Add the program for the communication to this task. Further information on task configuration can be found in the description of the function block [FB\_MBUSKL6781](ms-xhelp:///?Id=beckhoff-2b5d-4be4-b4d2-353673675260).

![51783759](/tcplclib_tc2_mbus/1033/Images/gif/6217536907__Web.gif)

## I/O configuration

Select the CX as target system and initiate a search for its hardware. In the project instance within the PLC section, you can see that the input and output variables are assigned to the corresponding tasks.

![63400648](/tcplclib_tc2_mbus/1033/Images/gif/6218355083__Web.gif)

Now link the global variables of PLC program with the inputs and outputs of the Bus Terminals. Create the Solution and enable the configuration.

When linking the data array, make sure that you select both the **All types** and **Continuous** options.   
Use the Shift key and the right mouse button to mark all data bytes of the terminal.

![33734921](/tcplclib_tc2_mbus/1033/Images/gif/6218370827__Web.gif)

After starting the PLC, the current values are regularly read by the counter.
